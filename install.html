Configuration de Git

Git est bien installé, vous allez pouvoir le configurer. Commencer par taper ces 3 lignes :

  • git config --global color.diff auto
  • git config --global color.status auto
  • git config --global color.branch auto

Ces commandes permettes juste de changer les couleurs

Nous allons maintenant insérer vos nom et email. Ce n'est pas obligatoire mais c'est pratique pour voir qui à fais les modifications.

  • git config --global user.name "votre_pseudo"
  • git config --global user.email moi@email.com

Nous avons finis pour la configuration de Git

Avant de passer à l'étape suivante, assurer vous bien d'avoir créer votre paire de clef et de l'avoir enregistrer dans git hub.

Je vous mets à disposition la doc de github pour windows concernant la clé ssh : clé SSH sur github

Créer et cloner le dépot

Dans votre terminal créer le répertoire de travail. Par exemple rubis :

  • mkdir rubis

Aller dans votre répertoire :

  • cd rubis/

Initialiser le depot Git :

  • git init

Cloner le dépot pour pouvoir le récupérer :

  • git clone git@github.com:EquipeK/Test.git

On privilégiera toujours de cloner via le protocole SSH car plus de sécurité. Et voilà vous avez récuperer tous ce qui faut pour travailler.

Nous allons maintenant voir comment deployer et récuperer le code

Faire des commits et envoyer son code

Après avoir fais des modifications dans votre code et que vous avez bien tester que tout fonctionnait bien, faites un commit. Un commit c'est quoi ? Dans les jeux vidéos on pourrais comparer un commit à un checkpoint. Avec git se serais prendre un snapshot de votre dépot. En cas d'erreur on peut revenir sur des commits précédents.

Je vais vous détaillez les étapes pour déployer votre code :

  • Une fois les modifications faites, avec la commande git status, on va lister les fichiers qui ont été modifié (ils seront de couleur rouge dans votre console).
  • Il faut ajouter ces fichiers au commit via le git add --all (l'option --all permet d'ajouter tous les fichiers en rouge). Si vous refaites un git status les fichiers seront en vert
  • les fichiers ajouter on fait un git commit -am "votre message". l'option -a permet d'associer le message du commit à tous les fichiers ajouter. l'optiom -m pour le message.
  • Avant de déployer son code, on met à jour son code en local avec celui qui est sur github : git pull --rebase . l'option --rebase permet de mettre vos commits au dessus des autres. Plus simple pour vous retrouver dans vos commit.
  • Une fois le git pull effectuer sans erreur, vous pouvez envoyer votre code : git push

Très important le message se trouvant entre les double quote DOIT etre parlant ET tous le monde doit comprendre de quoi vous parlez. Résumer en quelque mot votre commit. C'est très important sinon on s'y retouve plus. Imaginez 30 commit avec marqué "fix bug" ! bug de qui de quoi pourquoi ? un message concis et court.

Travailler avec des branch

On travail avec des branch quand on est en phase de développement et en phase de production. On aura deux branch une dev et l'autre prod/master avant de travailler vérifiez toujours sur quel branch vous travaillé. On NE travaille JAMAIS sur la branch MASTER !!!

On vérifie via la commande :

  • git branch

On change de branch via la commande

  • git checkout

Et voilà votre code est deployer !!

Un petit récapitulatif des commandes à suivre pour deployer son code

  • git status
  • git add --all
  • git commit -am "mon message que tout le monde comprend"
  • git pull --rebase origin master
  • git push origin master
